このプロジェクトは Claude Code CLI のセットアップディレクトリです。

## 技術スタック

- ツール: Claude Code CLI
- バージョン: 2.0.10
- OS: Windows 11

## コーディング規約

- インデント: スペース2個
- 命名規約: camelCase
- コメント: 日本語で記述

## よく使うコマンド

- Claude起動: `claude`
- バージョン確認: `claude --version`
- ヘルプ表示: `claude --help`

## 注意事項

- Claude Code CLIは対話型のAIアシスタントツールです
- 認証にはClaude Pro、Max、またはAnthropic Consoleアカウントが必要です

# 補助金申請システム - 様式2（経営計画書兼補助事業計画書）作成要件

## 1. 様式2の構成

様式2は以下のセクションで構成される：

### 【経営計画】セクション
1. **企業概要**
2. **顧客ニーズと市場の動向**

4. **経営方針・目標と今後のプラン**
5. **事業承継の計画**（事業承継加点希望者のみ）

### 【補助事業計画】セクション
1. **補助事業の内容**
   - 補助事業で行う事業名（30文字以内）
   - 販路開拓等（生産性向上）の取組内容
   - 業務効率化（生産性向上）の取組内容（任意）
   - 補助事業の効果

2. **経費明細表**（別セクション）

3. **資金調達方法**（別セクション）

## 2. 各セクションの記載要件

### 2.1 企業概要

**記載内容：**
- 事業の概要（創業年、主要サービス、顧客層）
- 事業別の売上高・売上総利益・売上シェア・客単価の表
- 立地場所の特性
- 主な商品・サービスのリスト（単価、営業利益率）
- 売上ランキング（売上総額TOP3、利益総額TOP3）
- 売上の状況（日次、週次、月次の来客数と売上パターン）
- 業務状況（従業員数、業務内容、課題）

**文体：** である調（常体）

**文字数制限：** 経営計画＋補助事業計画の合計で最大8ページ程度

**重要ポイント：**
- 数値を具体的に記載（概算値でも可）
- 課題を明確に記述（例：人件費、借入金返済、販路開拓の必要性）
- 新型コロナウイルス等の外部要因による影響も記載

### 2.2 顧客ニーズと市場の動向

**記載内容：**
- 市場全体の動向（業界統計、トレンド、出典明記）
- 自社事業に関わる市場分析
  - 地域別顧客分布（市町村別の売上構成比）
  - 人口に対する顧客数の割合
  - 未開拓地域の特定
- 事業別（店舗事業/オンライン事業等）の購入品特徴
  - 商品別売上構成比
  - 季節変動パターン
  - 客単価の分析

**文体：** である調（常体）

**重要ポイント：**
- 客観的データに基づく分析（出典を明記）
- 地域別の開拓余地を数値で示す

### 2.3 自社や自社の提供する商品・サービスの強み

**記載内容：**
- 競合他社と比較して優れている点

## 3. 経費明細表の要件

**記載内容：**
- 経費区分（①機械装置等費〜⑪委託・外注費）
- 内容・必要理由
- 経費内訳（単価×回数）
- 補助対象経費（税抜・税込を明記）

**計算項目：**
- (1) 補助対象経費小計（ウェブサイト関連費を除く）
- (2) 補助金交付申請額（(1)×補助率2/3、赤字事業者は3/4）
- (3) ウェブサイト関連費に係る補助対象経費小計
- (4) ウェブサイト関連費に係る交付申請額（(3)×補助率、(6)の1/4が上限）
- (5) 補助対象経費合計（(1)+(3)）
- (6) 補助金交付申請額合計（(2)+(4)）

**重要ポイント：**
- ウェブサイト関連費は全体の1/4以内
- 枠ごとに上限額が異なる（通常枠50万円、賃上げ枠200万円等）
- インボイス特例で50万円の上乗せ可能

## 4. 資金調達方法の要件

**左側：補助対象経費の調達一覧**

4. その他
5. 合計額（経費明細表(5)と一致）

**右側：「2. 補助金」相当額の手当方法**
- 2-1. 自己資金

# 様式3（経費明細表・資金調達方法）作成要件

## 1. 様式3の位置付け

様式3は**様式2の一部**であり、「補助事業計画書②【経費明細表・資金調達方法】」として記載する。

**ページ構成:**
- 様式2-①: 経営計画（最大8ページ）
- **様式2-②（様式3）: 経費明細表・資金調達方法（1-2ページ）**

## 2. 経費明細表（Ⅱ）の記載要件

### 2.1 基本構成

**表形式（単位：円）**

| 経費区分 | 内容・必要理由 | 経費内訳（単価×回数） | 補助対象経費（税抜・税込） |
|---------|--------------|-------------------|----------------------|

### 2.2 経費区分（11種類）

1. **①機械装置等費**
2. **②広報費** （チラシ、広告等）
3. **③ウェブサイト関連費** （ウェブサイト改修、ECサイト構築等）
4. **④展示会等出展費**
5. **⑤旅費**
6. **⑥開発費** （試作品開発、試供品製造等）
7. **⑦資料購入費**
8. **⑧雑役務費**
9. **⑨借料**
10. **⑩設備処分費**
11. **⑪委託・外注費**

### 2.3 記載項目

#### 各経費の記載内容

**必須項目:**
- **経費区分**: ①〜⑪のいずれか
- **内容・必要理由**: 何のために必要か（1-2行で簡潔に）
- **経費内訳**: 「単価×回数」形式で記載
- **補助対象経費**: 税込または税抜を明記

**記載例:**
```
経費区分: ②広報費
内容・必要理由: 新聞折り込みチラシ印刷費（1,000,000部×2回）・自社商品の広告のため
経費内訳: 132,000円×2回（税込）
補助対象経費: 240,000円
```

### 2.4 計算項目（重要）

**計算式とルール:**

1. **(1) 補助対象経費小計（ウェブサイト関連費を除く）**
   - ③以外の経費の合計

2. **(2) 補助金交付申請額（ウェブサイト関連費を除く）**
   - (1) × 補助率 2/3（赤字事業者は3/4）
   - 円未満切捨て

3. **(3) ウェブサイト関連費に係る補助対象経費小計**
   - ③ウェブサイト関連費の合計

4. **(4) ウェブサイト関連費に係る交付申請額**
   - (3) × 補助率 2/3（赤字事業者は3/4）
   - **(6)の1/4を上限（最大50万円）**
   - 円未満切捨て

5. **(5) 補助対象経費合計**
   - (1) + (3)

6. **(6) 補助金交付申請額合計**
   - (2) + (4)

**重要な制約:**
- **ウェブサイト関連費は全体の1/4以内**
- (4)が(6)の1/4以内であるかチェック必須
- 「いいえ」の場合は申請不可

### 2.5 補助率

**基本補助率: 2/3**
**赤字事業者: 3/4**

※赤字事業者 = 直近1期または直近1年間の課税所得金額がゼロ以下

### 2.6 枠ごとの上限額

| 枠 | 上限額 | 追加要件 |
|---|-------|---------|
| **通常枠** | 50万円 | なし |
| **賃金引上げ枠** | 200万円 | 事業場内最低賃金+30円以上 |
| **赤字事業者** | 200万円 | 課税所得ゼロ以下、補助率3/4 |
| **卒業枠** | 200万円 | 従業員数が小規模事業者の定義を超える |
| **後継者支援枠** | 200万円 | アトツギ甲子園ファイナリスト |
| **創業枠** | 200万円 | 過去3年以内に創業 |

### 2.7 インボイス特例

**上乗せ額: +50万円**

**要件:**
- 2021年9月30日〜2023年9月30日の課税期間で免税事業者であった
- 適格請求書発行事業者の登録を受けた

**注意:**
- 過年度事業で「インボイス枠」採択者は対象外

  const webExpenses = expenses.filter(e => e.category === '③ウェブサイト関連費');
  const otherExpenses = expenses.filter(e => e.category !== '③ウェブサイト関連費');

  // 3. 計算
  const calc1 = sumExpenses(otherExpenses); // (1)
  const calc2 = calc1 * (2/3); // (2) 補助率考慮
  const calc3 = sumExpenses(webExpenses); // (3)
  const calc4 = Math.min(calc3 * (2/3), calc6 * 0.25); // (4) 1/4制限
  const calc5 = calc1 + calc3; // (5)
  const calc6 = calc2 + calc4; // (6)

  // 4. バリデーション
  if (calc4 > calc6 * 0.25) {
    throw new Error('ウェブサイト関連費が全体の1/4を超えています');
