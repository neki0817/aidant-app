### ⚠️ 改善が必要な点

#### 1. **経費明細表・資金調達方法の欠落**
現状の様式2には以下が含まれていない：
- **経費明細表**（Ⅱ）
  - 経費区分（①機械装置等費〜⑪委託・外注費）
  - 内容・必要理由
  - 経費内訳（単価×回数）
  - 補助対象経費（税抜・税込）
  - 補助金交付申請額の計算（(1)〜(6)）

- **資金調達方法**（Ⅲ）
  - 補助対象経費の調達一覧
  - 補助金相当額の手当方法

**対策**: Phase 5の回答から経費明細表を自動生成する機能が必要

#### 2. **市場データの出典が未記載**
- 「リモートワーク普及により20%増加」などの記述に出典がない
- 審査時に信頼性が問われる可能性

**対策**:
- 業界統計データベースを構築
- 出典自動挿入機能の実装

#### 3. **売上推移の表形式が未実装**
参考資料（r3i_kisairei_coffee.pdf）では売上推移を表形式で記載：
```
| 項目 | 直近1期 | 前年同期 |
|------|---------|----------|
| 売上高 | 1,440万円 | 1,200万円 |
| 営業利益 | 240万円 | 180万円 |
```

**対策**: Phase 1データから自動的に表を生成

#### 4. **Phase 2-6の詳細情報の一部未活用**
シミュレーションで得られた以下の情報が様式2に反映されていない：
- 具体的なメニュー名（ブルーマウンテンシグネチャーケーキ等）
- 月間販売数（100ピース等）
- リピート率70%
- 来店時間帯の詳細

**対策**: プロンプト改善により、より詳細な情報を組み込む

## 5. 改善優先順位

### 最優先（Phase 5データから自動生成が必要）
1. **経費明細表の自動生成**
   - Phase 5の回答から経費区分を判定
   - 単価×回数の形式で記載
   - 補助金額を自動計算（2/3補助率、上限50万円等）

2. **資金調達方法の自動生成**
   - 自己資金と補助金の内訳を計算
   - 補助金入金までの資金手当方法を記載

### 高優先（プロンプト改善で対応可能）
3. **売上推移表の自動生成**
   - Phase 1データ（Q1-8, Q1-9）から表を作成
   - 前年比・成長率を自動計算

4. **Phase 2-6詳細情報の活用強化**
   - シミュレーションデータの具体例を漏らさず反映
   - メニュー名、販売数、リピート率等を組み込む

### 中優先（外部データ連携が必要）
5. **市場データ出典の自動挿入**
   - 業界統計APIとの連携
   - 地域別人口データの取得
   - 出典URLの自動付与
